网络升级

在区块 225,000，Yespower 将从共识中移除 — 此后 Dpowcoin 将完全采用 Argon2id。

背景：双重 PoW 时代

在区块 225,000 之前，Dpowcoin 使用共享同一目标值的两种独立工作量证明检查来验证每个区块头：hash = { Yespower(header), Argon2id(header) }，有效 ⇔ 两个哈希值均 ≤ target(nBits)。Yespower 作为低成本过滤器首先被验证；只有通过 Yespower 检查后才会计算 Argon2id。

为何移除 Yespower

由于两种算法共享同一目标值，Yespower 并未在 Argon2id 已提供的安全性之上增加任何实质保障，却使 LWMA 需要处理的工作量翻倍，并在没有任何可衡量收益的情况下使代码库复杂化。由于两项检查相互独立，移除 Yespower 不会影响难度调整或链权重 — 二者均与算法完全无关。双重 PoW 的想法值得一试，但这一实现所制造的问题比它解决的还多。“Dual PoW”这一名称仅因历史原因予以保留。

矿工须知

无需采取任何操作。由于 Yespower 和 Argon2id 始终共享同一目标值，基于 Argon2id 构建的挖矿软件在区块 225,000 前后均可保持不变地正常运行。轻量级 SPV 钱包可仅依赖 Argon2id 进行验证，因为 SHA-512 和 Argon2id 库几乎在所有平台上都可用。

重要声明

我们不使用“延迟工作量证明”（delayed PoW）技术，也与 Komodo 及其项目没有任何关联。在区块 225,000 之前，Dpowcoin 使用由 Yespower 和 Argon2id 组成的双重 PoW，其中 Argon2id 本身通过双重 SHA-512 加盐的两轮链式计算得出。
