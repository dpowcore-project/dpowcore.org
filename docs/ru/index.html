Обновление сети

На блоке 225,000 Yespower исключается из консенсуса — после этого Dpowcoin переходит исключительно на Argon2id.

Предыстория: эпоха Dual PoW

До блока 225,000 Dpowcoin проверял каждый заголовок двумя независимыми проверками proof-of-work с общей целью: hash = { Yespower(header), Argon2id(header) }, валиден ⇔ оба хеша ≤ target(nBits). Yespower проверялся первым как дешёвый фильтр; Argon2id вычислялся только в случае, если Yespower проходил проверку.

Почему Yespower исключается

Поскольку оба алгоритма использовали общую цель, Yespower не добавлял реальной безопасности сверх той, что уже обеспечивает Argon2id, при этом удваивая объём работы, который должен учитывать LWMA, и усложняя кодовую базу без какой-либо измеримой пользы для сети. Так как обе проверки независимы, исключение Yespower не влияет на корректировку сложности или вес цепи — оба параметра полностью не зависят от алгоритма. Идею Dual PoW стоило попробовать, но эта реализация создала больше проблем, чем решила. Название «Dual PoW» сохраняется исключительно по историческим причинам.

Примечание для майнеров

Никаких действий не требуется. Поскольку Yespower и Argon2id всегда использовали общую цель, майнинг-софт на базе Argon2id продолжает работать без изменений до и после блока 225,000. Лёгкие SPV-кошельки могут полагаться только на Argon2id для проверки, так как библиотеки SHA-512 и Argon2id доступны практически на любой платформе.

Важное примечание

Мы не используем технологию «delayed PoW» и никак не связаны с Komodo или её проектами. До блока 225,000 Dpowcoin использовал Dual PoW из Yespower и Argon2id, где сам Argon2id вычисляется как два последовательных раунда с солью из двойного SHA-512.
