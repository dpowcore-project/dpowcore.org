ネットワークアップグレード

ブロック225,000でYespowerがコンセンサスから削除されます — 以降、DpowcoinはArgon2id専用になります。

背景：デュアルPoW時代

ブロック225,000以前、Dpowcoinは1つのターゲットを共有する2つの独立したプルーフ・オブ・ワーク検証ですべてのヘッダーを検証していました：hash = { Yespower(header), Argon2id(header) }、有効 ⇔ 両方のハッシュ ≤ target(nBits)。Yespowerは低コストなフィルターとして最初に検証され、Yespowerを通過した場合のみArgon2idが計算されました。

Yespowerが削除される理由

両アルゴリズムが同じターゲットを共有していたため、YespowerはArgon2idがすでに提供しているセキュリティ以上の実質的な効果をもたらさず、LWMAが考慮すべき作業量を倍増させ、ネットワークに測定可能な利益をもたらすことなくコードベースを複雑化させていました。2つの検証は独立しているため、Yespowerの削除は難易度調整やチェーンウェイトに影響を与えません — どちらも完全にアルゴリズムに依存しません。デュアルPoWのアイデアは試す価値がありましたが、この実装は解決した以上の問題を生み出しました。「デュアルPoW」という名称は歴史的な理由のみで維持されています。

マイナーへの注意事項

特別な対応は必要ありません。YespowerとArgon2idは常に同じターゲットを共有していたため、Argon2idベースのマイニングソフトウェアはブロック225,000の前後で変更なく動作し続けます。SHA-512とArgon2idのライブラリはほぼすべてのプラットフォームで利用可能なため、軽量SPVウォレットはArgon2idのみに依存して検証できます。

重要な免責事項

当プロジェクトは「delayed PoW」技術を使用しておらず、Komodoまたはそのプロジェクトともいかなる関係もありません。ブロック225,000以前、DpowcoinはYespowerとArgon2idからなるデュアルPoWを使用しており、Argon2id自体は二重SHA-512でソルト化された2つの連鎖ラウンドとして計算されます。

Komodoとは無関係 — delayed PoWなし
